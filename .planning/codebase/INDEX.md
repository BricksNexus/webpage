## Project in One Paragraph

**BricksNexus** is a Next.js App Router app (JSX, no TypeScript) running alongside a legacy static HTML marketplace. Core flow: user submits a property address → `/api/property/enrich` pulls Census geocoder + NYC PLUTO + AttomData → `/api/opportunity/analyze` runs a 4-node LangGraph pipeline (Gemini 2.5 Flash) → structured Opportunity Report renders at `/opportunity-report`. A separate static HTML layer (`public/`) handles marketplace browsing, user auth (localStorage-based), contractor profiles, and opportunity/tokenization submission forms. Both surfaces share the same API routes.

## Key Conventions (TL;DR)

- **JSX only** — no TypeScript, never add `.ts`/`.tsx`
- **LLM calls** → `lib/llm-chat.js` only, never call Gemini directly
- **Property data** entry point → `lib/open-property/fetch-property-intel.mjs`
- **CORS** → always use `lib/api-cors.js` in API routes
- **New components** → `components/<subdomain>/` not root level
- **CSS** → Tailwind for Next.js pages; standalone `.css` files for static HTML pages
- **Secrets** → `.env.local` only (`ATTOMDATA_API_KEY`, `GOOGLE_API_KEY`, `GEMINI_MODEL`)

## Top Concerns (summary — see [CONCERNS.md](CONCERNS.md) for full list)

- **P0**: API keys potentially leaking via static `public/app-state.js`; no rate limiting on any route
- **P1**: Duplicate root-level files (`route.js`, `token-data.js`, etc.) that mirror `lib/` modules — dead code risk
- **P1**: No formal tests; quality gate is only `npm run build`
- **P2**: Static HTML auth is localStorage-only — trivially bypassable
- **P2**: LangGraph pipeline has no retry/timeout logic; single LLM failure breaks the entire report
